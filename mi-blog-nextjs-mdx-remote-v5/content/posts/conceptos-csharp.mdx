# operador Ternario ( ? :)

Este operador es un condicional abreviado SI ELSE  
```csharp
// Condición ? Si es Verdadero : Si es Falso
var resultado = (edad >= 18) ? "Adulto" : "Menor";
```
Dentro de C# un **null** dentro del lenguaje no es igual a un **null** que venga de la base de datos,  
de tal forma que para C# lo reconozca hay que converirlo, para saber si el valor en  la tabla es nulo  
se usa **DBNull.Value** por ejemplo en if normal tradicional seria:

```csharp
    if (reader["fecha1"] == DBNull.Value)
    {
        Fecha1 = null;
    }
    else
    {
        Fecha1 = Convert.ToDateTime(reader["fecha1"]);
    }
```

Con el operador ternario seria un poco mas abreviado:

```csharp
    Fecha1 = reader["fecha1"] == DBNull.Value ? null : Convert.ToDateTime(reader["fecha1"])
```

Ejemplo 2 de ternario
```csharp
// Usando el operador ternario tradicional (? :)
_connectionString = (configuration.GetConnectionString("DefaultConnection") != null) 
    ? configuration.GetConnectionString("DefaultConnection") 
    : throw new InvalidOperationException("...");
```
Nota si se cumple que DefaultConnection es nulla hay que repetir el codigo configuration.GetConnectionString("DefaultConnection")   
por eso la nueva forma moderna es usar **??** doble interrogacion , donde se le dice si lo de la izquierda no es nullo uselo  
en caso cotrario haga lo de la derecha 

```csharp
// Usando fusión nula (??) -> Más limpio y eficiente
_connectionString = configuration.GetConnectionString("DefaultConnection") ?? throw new InvalidOperationException("...");
```

# manejar tipo de datos entre Sql server y csharp

Por ejemplo si le pasamos los parametro un procedimiento almancenado de Sqlserver si especificamos el tipo de paramatro   
SqlServer realiza la consulta mas rapido ejemplo 1 y despues el mismo ejemplo pero con los tipos de datos: 

```csharp
 using var connection = _dbConnectionFactory.CreateConnection();
 using var command = new SqlCommand(sql, connection);

 command.Parameters.AddWithValue("@NitCed", nitCed);
 command.Parameters.AddWithValue("@Tipo", tipo);
```

la forma mejorada seria:
```csharp
 using var connection = _dbConnectionFactory.CreateConnection();
 using var command = new SqlCommand(sql, connection);

 command.Parameters.Add("@NitCed", SqlDbType.VarChar, 20).Value = nitCed;
 command.Parameters.Add("@Tipo", SqlDbType.VarChar, 3).Value = tipo;
```

    public static class SqlDataReaderExtensions
    {
        public static DateTime? GetNullableDateTime(this SqlDataReader reader, string columnName)
        {
            var ordinal = reader.GetOrdinal(columnName);
            return reader.IsDBNull(ordinal) ? null : reader.GetDateTime(ordinal);
        }
